# Curso de Buenas Prácticas y Código Limpio en C#

## Objetivo

Dar una base clara para escribir código legible, mantenible y consistente en C# y .NET.

## Qué son las buenas prácticas

Las buenas prácticas son hábitos y convenciones que ayudan a que el código sea más fácil de entender, probar y mantener.

## Qué es el código limpio

Código limpio es código que:

- se entiende rápido
- tiene nombres claros
- evita duplicación innecesaria
- mantiene una estructura simple
- facilita cambios futuros

## Requisitos previos

- Conocimiento básico de C# y .NET
- Uso básico de Git
- Visual Studio o Visual Studio Code
- .NET 6 o superior

## Terminología

- Buenas prácticas: estándares comprobados para resolver problemas comunes.
- Código limpio: estilo de escritura enfocado en claridad, mantenimiento y simplicidad.

## Principios clave

- Nombres descriptivos para clases, métodos y variables.
- Métodos pequeños con una sola responsabilidad.
- Evitar código duplicado.
- Separar responsabilidades por capas o módulos.
- Preferir composición sobre complejidad innecesaria.
- Escribir código fácil de leer antes que código “ingenioso”.

## Recomendaciones prácticas

- Usa formato consistente en todo el proyecto.
- Evita valores mágicos; usa constantes o configuraciones.
- Maneja errores de forma explícita.
- Valida entradas antes de procesarlas.
- Mantén comentarios solo cuando aporten contexto real.
- Revisa y refactoriza con frecuencia.

## Buenas señales

- El código se puede leer sin explicación adicional.
- Las funciones son cortas y predecibles.
- Los nombres reflejan intención.
- Es sencillo agregar pruebas.

## Mala señal

- Lógica mezclada en un solo método.
- Nombres ambiguos.
- Copia y pega repetida.
- Dependencias ocultas o innecesarias.
- Dificultad para modificar sin romper algo.

## Cierre

La meta no es escribir código perfecto, sino código que el equipo pueda mantener con confianza.
